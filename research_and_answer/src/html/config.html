照会回答パッケージプラグイン 設定

本プラグインは「依頼アプリ」と「回答アプリ」の2つのアプリにインストールして使うパッケージです。 このアプリがどちらの役割かを選択してください。フォーム項目のフィールドコード (依頼アプリ: questions テーブル・json・condition_json・related_links・related_rich_text・ 参照テーブル related / 回答アプリ: json・title・lookup・予備フィールド text_* / multi_text* / number_* / date_* / datetime_* / time_*)は、専用アプリテンプレートの規約に従う固定値です。

本プラグインは以下の2点を前提としています。いずれもkintoneの標準機能でアプリ管理者が別途 設定してください(プラグインは自動設定しません)。
・回答の追加は、依頼アプリのプロセス管理の「アクション」機能(アクションボタンで回答アプリへ レコードを自動生成)経由で行う想定です。
・回答アプリのレコードのアクセス権は、「回答部署」「依頼元部署」の組織に閲覧・編集権限を 付与する設定を想定しています。

回答を蓄積するアプリのアプリID(URLの /k/ 直後の数字)。関連レコード一覧の作成と 「集計・分析」ボタンの遷移先に使います。

レコード詳細画面でフォームプレビューを描画するスペースフィールドの要素IDです。

照会フォームを設計するアプリのアプリID。ルックアップ(照会の選択とフォーム定義のコピー)の 関連付け先に使います。

カスタマイズ形式(ページネーションあり)の一覧で、HTMLに <div id="virtual-table-div"></div> を含めてください。

選択した役割に必要なフィールド・スペース・一覧のうち、不足しているものだけを動作テスト環境に 追加します(既存の設定は変更しません)。リレーションは上で入力した相手アプリIDを使って 作成します。実行順: ①依頼アプリで実行し「アプリを更新」 → ②回答アプリで実行し 「アプリを更新」 → ③回答アプリでもう一度実行(依頼アプリに関連レコード一覧を自動追加) → 依頼アプリを「アプリを更新」。kintoneの参照整合性チェックは相手アプリの運用環境に対して 行われるため、この順が必要です(各ステップは画面の案内に従えばOKです)。
