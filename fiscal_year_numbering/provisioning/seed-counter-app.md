# カウンター専用アプリのセットアップ手順

`fiscal_year_numbering`プラグインは、採番の排他制御のために「カウンター専用アプリ」を1つ必要とする。
このアプリは環境(検証環境・本番環境)ごとに1度だけ作成する。

## スクリプトが自動でやること

```bash
node provisioning/seed-counter-app.js
```

1. カウンター専用アプリを新規作成する(`/k/v1/preview/app.json`)
2. 必要なフィールドを追加する(`key_sequence`, `combination_key`, `sequence_number`, `target_app_id`, `fiscal_year`, `era_code`, `era_year`, `segment_summary`)
3. 一般権限を「関係者全員: 閲覧+作成のみ許可(編集・削除は不可)」に設定する
4. 運用環境へデプロイする

認証は`X-Cybozu-Authorization`(リポジトリルートの`.env`の`KINTONE_USERNAME`/`KINTONE_PASSWORD`)を使う。
「アプリを作成するAPI」はAPIトークン認証を利用できない仕様のため、パスワード認証を使っている。

## 手動で行う必要があること(REST APIが提供されていないため)

- 実行後に表示されるアプリIDを`fiscal_year_numbering`の設定画面の「カウンター専用アプリID」に入力する。
- カウンター専用アプリを、番号を書き込む対象アプリと同じスペースに配置したい場合は、kintoneの画面から手動で移動する。
- アプリ名・アイコンなど見た目の調整をしたい場合は、kintoneの画面から手動で行う。

## 前提

- カウンター専用アプリは`kintone.api()`(ログインユーザー自身のセッション)からのみアクセスする設計であり、
  APIトークンは使用しない。
- 対象アプリ(番号を書き込む先)側の編集権限は「関係者は全レコード編集可」という広めの運用を前提にしている。
  この前提が崩れる場合(特定レコードのみ編集可、など)は設計の見直しが必要。
