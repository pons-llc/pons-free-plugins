# スプリットプラグイン

## 機能

指定した複数の記号・文字(または正規表現)で文字列フィールドの値を分割し、分割結果を複数の出力先フィールドへ
順番に格納する。分割結果の個数は入力値によって変動する(可変長)ため、出力先フィールドの数と一致しない
場合の扱いを以下の通り定めた。

- 分割設定(以下「設定行」)を複数持てる。1つの元フィールドに対して複数の設定行を作ることも、複数の異なる
  元フィールドに対してそれぞれ設定することもできる。
- 設定行ごとに、元フィールド(分割対象の文字列フィールド)・区切り指定・出力先フィールドの並び(配列)を持つ。

## 区切り指定(2モード、確定)

| モード | 指定内容 | 動作 |
| :-- | :-- | :-- |
| `CHARACTERS`(記号・文字) | 区切り文字の配列(例: `["-", "/", " "]`) | 配列内のいずれかの文字が現れるたびに分割する。各文字は正規表現の特殊文字として解釈されないよう、比較前に必ずエスケープする |
| `REGEX`(正規表現) | 正規表現パターン文字列(例: `[-/\\s]+`) | `String#split(new RegExp(pattern, 'g'))`で分割する |

## 出力先フィールドへの割り当て(可変長、確定)

分割結果の配列を先頭から順に出力先フィールドの配列へ割り当てる(`parts[0]`→`targetFieldCodes[0]`、
`parts[1]`→`targetFieldCodes[1]`、…)。

- 分割結果が出力先フィールドより**少ない**場合、余った出力先フィールドは空文字列でクリアする(前回保存時の
  古い値が残らないようにする、`wareki_date_format`と同じ上書き方針)。
- 分割結果が出力先フィールドより**多い**場合、はみ出した分は切り捨てる(結合して最後のフィールドに詰め込む
  等はしない。用途が不明な結合ロジックを増やすより、シンプルな「収まらない分は捨てる」方が予測しやすいと
  判断した。`判断記録.md`参照)。

## 発動タイミング(確定・明記)

**`app.record.create.submit` / `app.record.edit.submit` イベントでのみ分割処理を実行する。**

## 編集禁止の仕様(元メモ「挿入先の編集を禁止。元フィールドは一覧からの編集を禁止。」への対応、確定)

- **出力先フィールド**: レコード追加・編集画面(`app.record.create.show` / `app.record.edit.show`、
  モバイルは`mobile.app.record.create.show` / `mobile.app.record.edit.show`)で、設定されている
  すべての出力先フィールドの`disabled`を`true`にする(値の直接入力を禁止し、常にスプリット処理の結果のみが
  入る状態にする)。
- **元フィールド**: レコード一覧画面のインライン編集(`app.record.index.edit.show`。モバイルには
  インライン編集自体が存在しないため対象外)で、設定されているすべての元フィールドの`disabled`を`true`に
  する(一覧からの直接編集を禁止する)。レコード追加・編集画面(通常の詳細画面)では元フィールドは
  編集可能なまま(スプリットの入力元として使うため)。

## 設定画面

`kintone.plugin.app.setConfig()`にのみ保存する。

- 設定行を追加・削除できる。設定行ごとに:
  - 元フィールド(`SINGLE_LINE_TEXT`/`MULTI_LINE_TEXT`型から選択)
  - 区切りモード(記号・文字 / 正規表現)
  - 区切り文字の配列(`CHARACTERS`モード時、複数追加・削除できるテキスト入力)、または正規表現パターン
    (`REGEX`モード時、1つのテキスト入力)
  - 出力先フィールドの並び(複数追加・削除できる、順序が結果の割り当て順になる)
- 保存時に`js/lib/config-validation.js`でチェックし、不正な設定(正規表現として不正なパターンを含む)は
  保存させない。

## TDD

`src/js/lib/`配下の純粋ロジックはJestでユニットテストする(`pnpm test`)。

- `split.js` — 元の文字列値+区切り指定から分割結果の配列を返す。不正な正規表現は例外を投げず、
  分割せずに元の値1件だけの配列として返す(呼び出し側で毎回try/catchしなくても安全に使えるようにするため)
- `row-mapper.js` — 分割結果の配列+出力先フィールドの並びから、出力先フィールドの値オブジェクトを組み立てる
  (可変長の割り当てルールを実装する)
- `config-store.js` — `kintone.plugin.app.getConfig()`/`setConfig()`のペイロードの読み書きとデフォルト値
- `config-validation.js` — 設定(設定行の配列)のバリデーション。`REGEX`モードのパターンが
  `new RegExp()`でコンパイルできるかもここで検証する

kintone依存のグルーコード(`desktop.js`/`mobile.js`/`config.js`)は今回のタスクスコープでは実環境テスト
(Puppeteer)を行わない(別タスク、e2eは後回し)。
