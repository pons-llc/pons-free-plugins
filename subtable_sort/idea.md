# サブテーブルソートプラグイン

## 機能

サブテーブルの行を、複数の列を組み合わせた条件でソートする。`submit`時に自動でソートするか、
追加・編集画面のボタンで任意のタイミングでソートするかを選べる。任意実行の場合、「ソート済かどうか」を
示す文字列フィールドを設定でき、画面表示のたびに「未」にリセットし、ソート実行時に「済」にする。

- ソートルール(以下「ルール」)を複数持てる。ルールごとに、対象サブテーブル・ソートキー(複数)・
  発動タイミング・(任意実行時のみ)ソート済フィールドを持つ。

## ソートキー(複数列の組み合わせ、確定)

ソートキーを複数設定でき、先頭のキーを最優先に、Excelの複数列ソートと同じ考え方で順に比較する
(`js/lib/sort-comparator.js`)。キーごとに:

- 対象列(サブテーブル内のフィールドコード)
- 昇順(`ASC`)/降順(`DESC`)
- 値の型(`STRING`/`NUMBER`)。`NUMBER`は数値として比較し、`STRING`は文字列として比較する
  (日付・日時・時刻型は`STRING`のまま比較すればISO8601形式の辞書式順序が時系列順序と一致するため、
  型変換の手間がない)。

ソートは安定ソート(同順位の行は元の順序を保つ)。`Array.prototype.sort()`はES2019以降のJavaScript仕様で
安定性が保証されているため、追加の実装をせずに標準の`sort()`を利用する。

## 発動タイミング(2種類、確定)

| モード | 内部名 | 動作 |
| :-- | :-- | :-- |
| `submit`時 | `SUBMIT` | `app.record.create.submit`/`app.record.edit.submit`イベントで自動的にソートする |
| 任意実行 | `MANUAL` | レコード追加・編集画面のヘッダーメニュー領域にボタンを設置し、クリックしたときだけソートする |

`MANUAL`モードのボタンは`kintone.app.record.getHeaderMenuSpaceElement()`(PC専用)に描画するため、
`MANUAL`モードはPC専用とする(`判断記録.md`参照)。`SUBMIT`モードはイベントハンドラーのみで完結するため
PC・モバイル両対応。

## ソート済フィールド(`MANUAL`モードのみ、確定)

`MANUAL`モードのルールには、ソート済状態を示す文字列フィールド(サブテーブル外の通常フィールド)を
設定できる(元メモの「ソート済かどうかを挿入するフィールドを設定可能」)。

- 追加・編集画面を表示するたび(`create.show`/`edit.show`)に「未」へリセットする(元メモの
  「create,edit.showで未にして」に対応)。
- `MANUAL`ボタンでソートを実行すると「済」に変える(元メモの「任意実行時に済にする」に対応)。

「済」/「未」の文字列は固定とする(`js/lib/flag-values.js`)。

## モーダル確認プラグインとの組み合わせ(元メモの言及、確定・本プラグインでは何もしない)

元メモに「モーダル確認と組み合わせることでいい感じ」とあるが、本プラグインはあくまでソート実行と
ソート済フラグの更新のみを行う。「ソートを実行する前に確認ダイアログを出す」機能は`confirm_modal`
プラグイン(既存)側の責務とし、本プラグインは`confirm_modal`と技術的に連携する仕組み(共通のイベント等)を
持たない(2つのプラグインは完全に独立したまま、同じアプリに両方インストールして使うことを想定した組み合わせ、
`判断記録.md`参照)。

## 設定画面

`kintone.plugin.app.setConfig()`にのみ保存する。

- ルールを追加・削除できる。ルールごとに:
  - 対象サブテーブル
  - ソートキー(サブテーブル内フィールド・昇順/降順・値の型を複数追加・削除できる)
  - 発動タイミング(`submit`時/任意実行)
  - ソート済フィールド(`MANUAL`選択時のみ表示、テーブル外の文字列フィールドから選択)
- 保存時に`js/lib/config-validation.js`でチェックする(サブテーブル未選択、ソートキー0件等)。

## TDD

`src/js/lib/`配下の純粋ロジックはJestでユニットテストする(`pnpm test`)。

- `sort-comparator.js` — ソートキーの配列から比較関数を組み立て、サブテーブルのvalue配列をソートした
  新しい配列を返す(`STRING`/`NUMBER`の型比較、複数キーの優先順位、昇順/降順を含む)
- `config-store.js` — `kintone.plugin.app.getConfig()`/`setConfig()`のペイロードの読み書きとデフォルト値
- `config-validation.js` — 設定(ルールの配列)のバリデーション

kintone依存のグルーコード(`desktop.js`/`mobile.js`/`config.js`、ボタンのDOM描画・
`kintone.app.record.getHeaderMenuSpaceElement()`呼び出しを含む)は今回のタスクスコープでは実環境テスト
(Puppeteer)を行わない(別タスク、e2eは後回し)。

type, value } } }`)、`kintone.app.record.getHeaderMenuSpaceElement()`の利用可能画面、`kintone.app.record.set()`
での部分更新の仕様を確認済み。セキュアコーディングガイドラインでのリスクチェックは
`security-checklist.md`を参照。
