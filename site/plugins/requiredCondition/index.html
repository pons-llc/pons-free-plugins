条件つき必須プラグイン

ドロップダウンやラジオボタンフィールドが特定の値のときだけ、指定した別のフィールドを必須項目にするプラグインです。 「区分」が「その他」の場合だけ「詳細」を必須にする、といった条件付きの入力チェックを、フィールドを常に必須にせず実現できます。

入力制御 #条件付き必須 #入力チェック
ダウンロード(v1.0.0)

ユースケース

「区分」ドロップダウンで「その他」を選んだ場合のみ「詳細」フィールドを必須にし、それ以外の値のときは必須にしません。レコードの保存(作成・編集)時に、条件を満たしているのに対象フィールドが未入力だとエラーになります。

条件値ごとに必須対象フィールドを複数選択できるため、「承認区分」が「却下」のときに「却下理由」と「却下日」の両方を必須にする、といった設定も1行でまとめられます。

セットアップ手順

  1. プラグインをアプリに追加し、設定画面を開く。
  2. 「条件フィールド追加」ボタンで、必須となる条件のもとになるフィールド(ドロップダウン・ラジオボタン)を選択する。
  3. 「条件値を追加」ボタンで、条件となる値と、必須対象とするフィールドを複数選択で指定する。
  4. 「設定を保存」を押し、アプリを更新する。

必須チェックはレコードの保存(作成・編集画面のsubmit)時にのみ行われます。一覧画面のインライン編集は対象外です。

セキュリティレビュー結果サマリ

kintoneセキュアコーディングガイドラインに沿って実装時にチェックしています。特に重要な点は次のとおりです。

  • 設定画面でフィールドコード・選択肢を表示する箇所は、公開前レビューでinnerHTMLからtextContentに修正し、アプリ管理者が設定した文字列がそのままHTMLとして解釈されないようにしています。
  • REST APIの利用はフォームフィールド一覧の取得のみで、kintone自身への呼び出しに限定しています。外部サーバーへの通信は一切行いません。
  • 開発当初、外部のライセンス認証サーバーへ通信するコードが混入していましたが、公開前レビューで発見し削除しています。

API実行数について

このプラグインは設定画面を開いたときに1回だけkintone.api()でフォームフィールド一覧を取得します(kintone自身への呼び出し)。 レコード保存時の必須チェック自体はローカルなJavaScript処理のみで完結し、APIは実行しません。
